# Phase 10 CI パイプライン構築

## 目的・前提・方針案

Phase 10 のうち CI のみを対象とする（CD は別タスク）。  
`.github/workflows/ci.yml` を新規作成し、コードの品質ゲートを自動化する。  
GitHub Actions マネージドランナー上で完結（AWS リソース・ECR push なし）。

## 計画

### Phase 1: `.github/workflows/ci.yml` 作成

- [x] トリガー設定: `push` / `pull_request`（対象: main）、全 job 並列実行
- [x] `lint-backend` job: `uv run ruff check .` + `uv run mypy app/`（working-dir: backend/、DB 不要）
- [x] `test-backend` job: pytest（service: pgvector/pgvector:pg16、env: TEST_DATABASE_URL / DATABASE_URL / S3_BUCKET をダミー値で設定）
- [x] `lint-frontend` job: `npm ci` + `npm run lint`（working-dir: frontend/、env 不要）
- [x] `docker-build` job: backend/frontend の prod イメージをビルド検証（`--platform linux/amd64`、frontend は `--build-arg NEXT_PUBLIC_API_URL=http://placeholder`、ECR push なし）
